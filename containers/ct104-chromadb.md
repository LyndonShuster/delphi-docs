# CT 104 - ChromaDB

## Purpose
Vector store for semantic document retrieval. Stores document embeddings
generated by nomic-embed-text and answers similarity queries at inference
time. Has no knowledge of models, documents, or application logic —
it stores and retrieves vectors only.

## Container Spec
    VMID:         104
    Hostname:     chromadb
    Cores:        2
    RAM:          2048MB
    Swap:         512MB
    Disk:         10GB
    IP:           192.168.x.56/24
    Unprivileged: yes
    Features:     nesting=1

Disk allocation scales with corpus size. 10GB is sufficient for
thousands of documents. Increase if ingesting large corpora.

## Installation

Enter the container:
    pct enter 104

Update and install dependencies:
    apt update && apt upgrade -y && apt install -y python3 python3-pip python3-venv curl

Create the venv and install ChromaDB:
    python3 -m venv /opt/chromadb-venv
    source /opt/chromadb-venv/bin/activate
    pip install chromadb

Create the data directory:
    mkdir -p /var/lib/chromadb

## Startup Script

    cat > /opt/start-chromadb.sh << EOF
    #!/bin/bash
    source /opt/chromadb-venv/bin/activate
    chroma run --host 0.0.0.0 --port 8000 --path /var/lib/chromadb
    EOF

    chmod +x /opt/start-chromadb.sh

## Systemd Service

    cat > /etc/systemd/system/chromadb.service << EOF
    [Unit]
    Description=ChromaDB Vector Store
    After=network.target

    [Service]
    Type=simple
    User=root
    ExecStart=/opt/start-chromadb.sh
    Restart=always
    RestartSec=3

    [Install]
    WantedBy=multi-user.target
    EOF

    systemctl daemon-reload
    systemctl enable --now chromadb

## API Notes
ChromaDB 1.5.x uses a v2 API. Paths differ from older versions
and most tutorials found online.

Heartbeat endpoint:
    /api/v2/heartbeat

Collections endpoint requires tenant and database parameters:
    /api/v2/tenants/default_tenant/databases/default_database/collections

When using the Python client these paths are handled automatically.
Always use the Python client from CT 105 rather than raw HTTP calls.

When passing documents to ChromaDB always provide embeddings explicitly.
ChromaDB will attempt to generate its own embeddings if none are provided,
requiring onnxruntime and tokenizers packages. The stack uses
nomic-embed-text via Ollama for all embeddings instead.

Correct upsert pattern:
    collection.upsert(
        documents=documents,
        embeddings=embeddings,
        metadatas=metadatas,
        ids=ids
    )

Correct query pattern:
    collection.query(
        query_embeddings=[query_embedding],
        n_results=5
    )

## Collections
The agent pipeline uses one collection by default:
    corpus      active RAG corpus, populated by the ingest pipeline

Collections are created automatically on first use by the agent.
No manual collection setup is required.
